1. Problem Statement

The Core Issue

The current Customer Help Center / SAC at support.atlassian.com is a search-first, fragmented experience that forces customers to navigate multiple disconnected touchpoints to resolve issues.

Current Fragmented Touchpoints

TouchpointLocationProblem
In-Page HelpEmbedded in product pagesLimited to static links; no AI-powered contextual assistance
Ask Rovo ButtonHeader icon (current)Opens a slider panel — not prominent, not unified with search
Header Search BarHeaderShows content search results only; confused with “Contact us”
Loom HelpSeparate integrationDisconnected from support ticket creation workflow
Live Chatsupport.atlassian.comRequires navigating away from product; no product context passed
Support Ticket / Contact Formsupport.atlassian.com3+ hops to reach; no AI deflection before submission

Key Pain Points

  • 3+ hops to resolution: Customers navigate through multiple pages and UI elements before finding relevant help
  • Noisy cross-product results: Search returns loosely relevant articles from across all products, requiring heavy manual filtering
  • Search confused with “Contact us”: Users cannot distinguish between self-service search and the escalation path to agents
  • AI not front-and-center: Existing AI capabilities (Help AI Chat, AI search) are buried behind secondary UI elements, not the primary entry point
  • No contextual awareness: When users do reach support, they must re-explain their context (what page they were on, what they were trying to do)
Research Insight

Customers now expect AI assistance as a default, valuing conversational help that: understands intent in natural language, guides with clear step-by-step answers, and surfaces only the most relevant content instead of “piles of documents.”

Without a shift from search-first to AI-first, the Help Center will lag customer expectations and miss the opportunity to: increase self-help engagement, reduce unnecessary Contact App traffic, and build a consistent, reusable AI Help Center that can power multiple domains across Atlassian via CSM and Rovo.

2. Hypothesis

Core Hypothesis

By making Rovo / AI the primary, unified entry point via a single omni-box on the header — where users can search content with AI summaries, get support help, access live chat, raise support tickets, record Loom videos for context, see page-level help, and interact with suggested prompts — we will:

  • Increase self-help engagement by providing instant AI-powered answers before users need to contact an agent
  • Reduce Contact App / ticket volume through AI deflection and autonomous resolution
  • Deliver the conversational, AI-first experience customers expect from modern support tools
  • Decrease time to resolution by passing full page context, conversation history, and Loom recordings when escalation is needed
  • Build a reusable AI Help framework that can power multiple domains across Atlassian via CSM and Rovo

A multi-state component that serves as the single entry point for all support interactions, living in the application header.

State 1 — Collapsed Header Bar

“Ask Rovo – Search / Get In-Product Help & Support” text field with action icons (attach, mic, sparkle/AI). Always visible in the header. Clicking or focusing on it triggers the expanded popup.

5. User Flows

15 flows (A through O)

All flows originate from the unified omni-box on the Support Insights 360 starter page. Each flow describes the user journey, UI state transitions, and expected outcome.

Flow A — AI Search with Summary

User types a query in the omni-box → Popup transforms to State 3 (Search Results with AI Summary) → AI Summary card appears at top with Rovo icon and structured answer → Relevant Search Results listed below → Clicking a result navigates to the content page.

Example: User types “permissions for jira” → AI Summary: “Permission schemes in Jira define what users can do within a project. They control access to features like creating, editing, and managing issues. You can configure them under Project Settings > Permissions.” → Results: Getting Started with Support Insights 360, Configuring Data Sources, What are permission schemes in Jira?, Jira Service Management Documentation.

Reference: Mock 4

Flow B — In-Page Help

Omni-box popup displays contextual In-Page Help links based on the current page URL. The page URL (www.williamsracing/support-insights-360) is auto-detected and shown as a dismissible chip in the context bar.

Links shown: “Getting Started” | “Support Best Practices” — contextual to the Support Insights 360 dashboard.

Clicking an In-Page Help link opens the relevant help article in a new tab or inline preview.

Flow C — Loom Video Recording for Support Ticket

User clicks “Create a Support Ticket” in the action row → Ticket creation form appears with an option to “Record a Loom” → Loom SDK integration opens the recorder (screen + camera) → User records the issue contextually while on the Support Insights 360 page → Recording attaches to the new support ticket form → Ticket submitted with video context, page URL, and any prior Rovo conversation as background.

Integration: Loom SDK (existing product integration, not native browser recording).

Flow D — Live Chat with Bot-First Deflection

User clicks “Live Chat” in the action row → Rovo AI greets and classifies intent → AI attempts autonomous resolution (FAQ answers, account actions, guided steps) → If unresolved, handoff to human agent with full transcript, page context, and user metadata.

Sub-flow D.a — With Page Context

Page URL chip is included. Agent sees the user was on Support Insights 360, viewing SLA breach data.

Sub-flow D.b — Without Page Context

User dismisses the context chip before initiating chat. Agent receives conversation only.

During queue wait: show estimated wait time, offer callback option, suggest “While you wait, try asking Rovo…”

Flow E — Suggested Prompt to Rovo Side Panel

User clicks a suggested prompt from “Try Asking” (e.g., “What are permission schemes in Jira?”) → Rovo side panel opens (State 4) → Conversational AI session begins with that prompt pre-filled → Rich AI response with summary, visualizations, and follow-up suggestions.

Flow F — Recently Asked / Persistent Conversations

User sees their recent questions in the “Recently Asked” section (e.g., “Why is the EMEA billing queue surging?”, “Show SLA breach rate for NA Chat”) → Clicking one re-executes the query in the omni-box.

If a previous live chat was in progress, user can resume the conversation asynchronously (persistent chat). The conversation thread is preserved across sessions.

Flow G — AI-to-Agent Escalation with Full Context

AI cannot resolve the user’s issue → User is offered “Talk to an Agent” → Full conversation transcript + page context (Support Insights 360 URL) + Loom video (if recorded) + CRM profile auto-attached → Personalized AI routing selects best-fit agent based on intent, sentiment, skill match, and customer priority → Agent receives screen pop with all context. Zero re-explanation needed.

Flow H — Channel Switch (Chat → Voice/Video)

During live chat, user or agent can escalate to voice or video call without losing context. Full chat transcript, page context, and attached media carry over seamlessly. User does not need to repeat any information.

Flow I — Proactive / Predictive Alerts

The omni-box proactively surfaces known issues, outages, or incidents contextually before the user even asks. Inspired by Intercom Fin proactive triggers, ServiceNow Predictive Intelligence, and Statuspage integration.

I.1 — Incident / Outage Alert

Statuspage detects an ongoing incident affecting Jira Cloud in the user’s region → Omni-box header shows a warning badge/indicator → On open, a prominent alert banner appears: “We detected an ongoing incident affecting Jira Cloud in EMEA. Status: Investigating. ETA: ~30 min.” with a link to the Statuspage incident → User can dismiss, subscribe to updates, or ask Rovo “Is this affecting my project?”

I.2 — SLA/KPI Threshold Breach (Support Insights 360 context)

Rovo detects that the SLA Breach Risk on the current page has exceeded the 5% target (currently 8.2%) for the third consecutive day → Omni-box proactively shows: “SLA breach rate at 8.2% — exceeding your 5% target for 3 days. Want me to analyze root causes?” → Clicking opens the Rovo side panel with pre-filled analysis.

Maps to the left sidebar alerts: P1/P2 Tickets Surge · high Resolution Time Spike · medium SLA Breach Risk · high

I.3 — Page Dwell Time Trigger

User has been on the Support Insights 360 page for >3 minutes without interaction → Omni-box subtly pulses or shows a helper tooltip: “Need help understanding this dashboard? Try: ‘What are the top 3 drivers of ticket volume this week?’”

I.4 — Repeated Error Pattern

User has attempted the same action 3+ times (e.g., filter configuration failing) → Omni-box proactively suggests: “Having trouble with filters? Here’s a guide: Configuring Data Sources” with a direct link.

I.5 — AI-Predicted Churn Risk

Backend ML model identifies the user as at-risk (decreased usage, multiple unresolved tickets, low CSAT) → Omni-box shows: “We noticed you’ve had a few open issues. Would you like to connect with a dedicated support specialist?” → Clicking opens Live Chat with priority routing.

Flow J — Self-Service Actions

User requests an action in the omni-box (e.g., “reset my API token”, “check my instance health”, “clear cache”) → Rovo executes the action autonomously via backend APIs → Confirmation shown in the omni-box. No agent involvement needed.

Flow K — Post-Interaction Feedback

After resolution (AI or human), a CSAT/NPS micro-survey appears inline in the omni-box. Thumbs up/down on AI answers with “Not helpful? Tell us why” → Feedback feeds into AI model retraining and content gap detection. If marked unhelpful, auto-escalates to human agent.

Flow L — Open Case Tracker

User can view and track status of all open support tickets directly from the omni-box without navigating to support.atlassian.com. Inspired by Salesforce Case Portal and ServiceNow My Requests.

L.1 — View Open Cases

User clicks “View History” in the omni-box toolbar → “My Cases” panel shows a list of open tickets with: ticket ID, subject, status (Open / In Progress / Waiting on Customer / Resolved), priority badge, assigned agent name, last updated timestamp, SLA countdown if applicable. Sorted by most recent activity.

L.2 — Case Detail and Actions

User clicks a specific ticket → Expanded view shows full case timeline: original submission (including Loom video if attached), agent responses, AI-generated summary of the conversation so far. User can: add a comment/reply, attach a new file or Loom recording, escalate priority, or mark as resolved.

L.3 — Reopen a Resolved Case

User sees a recently resolved ticket → Clicks “Reopen” → Adds context on why the issue recurred → Case is re-opened and routed back to the same agent (affinity routing) with full history preserved.

L.4 — Case Status Notifications

When a tracked case gets an update (agent replied, status changed, resolved), a notification badge appears on the omni-box header → Opening the omni-box shows the notification inline: “Your ticket #12345 ‘Billing discrepancy’ has been updated — Agent replied 5 min ago” → Clicking navigates to the case detail view.

L.5 — Create New Ticket from Tracker

From the case tracker view, user can click “Create a Support Ticket” to start a new case → Form pre-populates with page context (Support Insights 360 URL) and any recent Rovo conversation history as background context.

Flow M — Rich AI Response with Data Visualization

User asks a data-oriented question (e.g., “What’s driving the Billing ticket surge?”) → Rovo side panel opens → AI responds with structured content: Summary paragraph, data visualization (bar chart showing ticket distribution by category), Key Findings bullet list, Recommendations as actionable items → User interacts via thumbs up/down, copy, or refresh → “Try following up with:” shows contextual next questions.

Reference: Live prototype Rovo side panel

Flow N — Quick Category Intent Selection

User opens Rovo help → Sees “I’m Rovo. How can I help?” with quick category buttons (Billing issue Account access Product setup Technical problem) → Clicking a category pre-classifies intent and starts a targeted conversation or routes to the appropriate help path.

Reference: Live prototype bottom panel

Flow O — In-Page AI Follow-ups

On the Support Insights 360 dashboard, Rovo generates “Today’s Summary” with AI-powered insights. Below the summary, “Ask Rovo a follow-up” prompts appear contextual to the page data (e.g., “What’s driving the Billing ticket surge?”, “Analyze NA Chat SLA breach risk”). Clicking a follow-up prompt opens the Rovo side panel with that question pre-filled, maintaining full page context.

6. User Stories & Acceptance Criteria

End User / Customer Stories

US-01: As a user, I want to search for help using natural language so that I get an AI-summarized answer instantly.
GIVEN I am on the Support Insights 360 page and click the omni-box
WHEN I type “permissions for jira” and press Enter or click Send
THEN the popup shows an AI Summary card at the top with a structured answer, followed by a list of relevant search results I can click to navigate to the full article
US-02: As a user, I want contextual in-page help links so that I can access relevant documentation without searching.
GIVEN I am on the Support Insights 360 page and open the omni-box
WHEN the popup loads
THEN I see the current page URL as a context chip and In-Page Help links (“Getting Started”, “Support Best Practices”) relevant to this specific dashboard
US-03: As a user, I want to record a Loom video showing my issue and attach it to a support ticket.
GIVEN I have clicked “Create a Support Ticket” in the omni-box
WHEN I click “Record a Loom”
THEN the Loom SDK recorder opens, I can record my screen, and upon completion the recording is automatically attached to the ticket form along with the page context URL
US-04: As a user, I want to start a live chat where AI tries to help me first before connecting to a human agent.
GIVEN I click “Live Chat” in the omni-box action row
WHEN the chat session starts
THEN Rovo AI greets me, asks about my issue, and attempts to resolve it autonomously. If it cannot, it offers to connect me to a human agent with my full conversation and page context auto-attached.
US-05: As a user, I want to click a suggested prompt and have Rovo answer in a side panel conversation.
GIVEN I see “Try Asking” suggestions in the omni-box popup
WHEN I click “What are permission schemes in Jira?”
THEN the Rovo side panel opens with a conversational AI response, including structured content, and I can ask follow-up questions
US-06: As a user, I want to see my recent questions and resume where I left off.
GIVEN I have previously asked questions via the omni-box
WHEN I open the omni-box popup
THEN I see “Recently Asked” with my past questions, and clicking one re-executes the query with current context. If a live chat was in progress, I can resume it.
US-07: As a user, I want proactive alerts about incidents affecting my products.
GIVEN there is an active Statuspage incident affecting Jira Cloud in my region
WHEN I view any Atlassian product page
THEN the omni-box header shows a warning badge, and opening it displays a prominent alert with incident status, ETA, and options to subscribe to updates or ask Rovo for details
US-08: As a user, I want to view and manage my open support tickets without leaving my current page.
GIVEN I have open support tickets
WHEN I click “View History” in the omni-box
THEN I see a “My Cases” panel with all my open tickets showing status, priority, agent, last update, and SLA countdown. I can click into any ticket to view details, add comments, or reopen a resolved case.
US-09: As a user, I want to perform self-service actions directly from the omni-box.
GIVEN I type “reset my API token” in the omni-box
WHEN Rovo processes my request
THEN Rovo executes the action via backend API and shows a confirmation message in the omni-box. No agent involvement.
US-10: As a user, I want seamless escalation from AI to a human agent without repeating context.
GIVEN Rovo AI could not resolve my issue after a conversation
WHEN I click “Talk to an Agent”
THEN my full conversation transcript, page URL, Loom recording (if any), and CRM profile are auto-attached to the agent handoff. The agent receives a screen pop with all context and I do not need to re-explain anything.
US-11: As a user, I want to provide feedback on AI answers to help improve them.
GIVEN Rovo has provided an AI answer
WHEN I click thumbs down and select “Not helpful”
THEN I am prompted to explain why, the feedback is logged for model retraining, and I am offered escalation to a human agent
US-12: As a user, I want to receive case status notifications directly in the omni-box.
GIVEN I have an open support ticket
WHEN an agent replies or the status changes
THEN a notification badge appears on the omni-box header, and opening it shows the update inline with a link to the full case detail
US-13: As a user, I want to switch from chat to voice/video without losing context.
GIVEN I am in a live chat session
WHEN I or the agent initiate a voice/video escalation
THEN the call connects with full chat transcript and context carried over; I do not need to repeat any information
US-14: As a user, I want to select a quick help category to get targeted support faster.
GIVEN I open the Rovo help panel
WHEN I click “Billing issue”
THEN Rovo begins a targeted conversation about billing, pre-classifying my intent for faster routing to a billing specialist if escalation is needed
US-15: As a user, I want dashboard-embedded AI follow-up prompts so I can ask questions about page data in one click.
GIVEN I am viewing the Support Insights 360 “Today’s Summary”
WHEN I click “What’s driving the Billing ticket surge?”
THEN the Rovo side panel opens with a rich response including summary, data visualization (bar chart), key findings, and recommendations, with follow-up suggestion chips

7. Edge Cases & Error Handling

ScenarioHandling
No search results foundShow “No results found. Try rephrasing your question or contact support.” Offer Live Chat and Create Ticket buttons.
AI summary generation fails / times outFall back to traditional search results. Show subtle error: “AI summary unavailable. Here are matching articles.”
Loom SDK fails to initialize or user denies screen permissionsShow error message with manual file upload fallback. “Unable to start recording. You can attach a file instead.”
Live chat agents unavailable (all busy)Show estimated wait time, offer callback option, suggest Rovo AI for immediate help.
Page context cannot be detected (generic pages)Context chip shows “No page context detected” with option to manually enter URL.
User is unauthenticatedShow login prompt before allowing AI search, ticket creation, or live chat. Read-only KB results still accessible.
Network connectivity issues mid-flowQueue messages locally, retry on reconnect. Show “Connection lost. We’ll retry when you’re back online.”
Channel switch fails (voice/video unavailable)Show “Voice/video not available in your environment. Continue via chat.”
Self-service action fails or requires elevated permissionsShow clear error: “This action requires admin permissions. Would you like to request access?”
AI misroutes to wrong agent skill groupAgent can re-route with one click. Customer sees “Connecting you to the right specialist…”
Multilingual detection failsDefault to English with language selector for manual override.
Proactive trigger fires at inappropriate timeTrigger has a cooldown period (no re-trigger for 30 min). User can dismiss and suppress for session.
Persistent conversation data expiredShow “Previous conversation no longer available. Start a new one?”
CRM / context lookup times out during routingRoute to general queue with graceful degradation. Agent can pull context manually.
Concurrent session limit reached for agentCustomer stays in queue with updated wait time. Offer callback or AI deflection.

Personalized AI Routing (from omni-box to agent)

  • Customer context ingestion: CRM profile, interaction history, sentiment score, LTV, preferred language & channel
  • Intent & sentiment detection: NLP/LLM classifies reason for contact and emotional tone in real time as user types
  • Skill-based matching: AI maps intent + context to best-fit agent skills, certifications, availability
  • Personalized queue priority: High-value/at-risk customers prioritized; repeat callers routed to last agent (affinity routing)
  • Channel-aware routing: Route to voice, chat, email, WhatsApp based on preference and urgency
